# RAG Chatbot

A Retrieval-Augmented Generation (RAG) chatbot that answers questions grounded in your own documents. Supports multi-turn conversation with full chat history. Built with FAISS, LangChain, Claude, FastAPI, and Streamlit.

## How it works

1. PDFs are loaded, split into chunks, and embedded using `sentence-transformers/all-MiniLM-L6-v2`
2. Embeddings are stored in a local FAISS vector index
3. On each question, the top-3 most relevant chunks are retrieved by similarity search
4. The chunks plus the full conversation history are passed as context to Claude (`claude-sonnet-4-6`), which generates a grounded answer

## Features

- **Grounded answers** — Claude only answers from document context; says "I don't know" if the answer isn't there
- **Multi-turn chat** — conversation history is passed with every request so follow-up questions work naturally
- **Source provenance** — each answer shows the retrieved document chunks it was based on
- **Chat UI** — Streamlit interface with a proper chat layout
- **REST API** — FastAPI `/ask` endpoint for programmatic access

## Project structure

```
rag-chatbot/
├── app/
│   ├── ingest.py         # Load PDFs, chunk, embed, and save FAISS index
│   ├── retriever.py      # Load FAISS index and run similarity search
│   ├── chain.py          # Build prompt (with history) and call Claude API
│   └── api.py            # FastAPI server with /ask endpoint
├── ui/
│   └── streamlit_app.py  # Streamlit chat UI with session state
├── data/                 # PDF documents
├── embeddings/           # FAISS index (generated by ingest.py)
├── .streamlit/
│   └── config.toml       # Streamlit config (disables noisy file watcher)
└── requirements.txt
```

## Setup

**1. Create and activate a virtual environment**

```bash
python -m venv .venv
source .venv/bin/activate
```

**2. Install dependencies**

```bash
pip install -r requirements.txt
```

**3. Set your Anthropic API key**

Create a `.env` file in the project root:

```
ANTHROPIC_API_KEY=your_key_here
```

**4. Add your PDFs**

Place PDF files in the `data/` directory.

**5. Build the FAISS index**

```bash
python app/ingest.py
```

This chunks your PDFs, embeds them, and saves the index to `embeddings/faiss_index/`.

## Running the app

**Streamlit UI**

```bash
streamlit run ui/streamlit_app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser. The UI keeps a running chat history for the session — follow-up questions have full context of previous turns.

**FastAPI server**

```bash
cd app && uvicorn api:app --reload
```

Send a question:

```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the five pillars of the AWS Well-Architected Framework?"}'
```

Interactive API docs are available at [http://localhost:8000/docs](http://localhost:8000/docs).

## Stack

| Layer | Tool |
|---|---|
| PDF loading | LangChain + PyPDF |
| Chunking | `RecursiveCharacterTextSplitter` (1000 chars, 50 overlap) |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` |
| Vector store | FAISS |
| LLM | Anthropic Claude (`claude-sonnet-4-6`) |
| API | FastAPI + uvicorn |
| UI | Streamlit |
